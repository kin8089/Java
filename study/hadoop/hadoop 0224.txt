-- 빅데이터

기존 데이터보다 방대하여 기존 방법으로 수집/저장/분석이 어려운 정형 및 비정형 데이터를 의미

빅데이터는 테라 이상의 데이터라고 정의하기도 하며 대용량 데이터를 처리하는 아키텍처라고도 정의


특징

1. 크기 (Volume) : 일반적으로 수십 테라바이트 이상 처리

2. 속도 (Velocity) : 데이터를 빠르게 처리하고 분석하는 속성

3. 다양성 (Variety) : 다양한 데이터를 의미하며 정형화 종류에 따라 정형, 비정형, 반정형데이터로 분류

4. 정확성

5. 가변성 : 데이터가 맥락에 따라서 의미가 달라짐

6. 시각화 : 데이터를 수집하여 용도에 맞게 정보를 가공하는 과정을 거쳐 정보의 사용자가 쉽게 이해 가능



플랫폼 : 빅데이터 기술의 집합체이자 기술을 잘 사용할 수 있도록 준비한 환경

	기업들은 빅데이터 플랫폼을 통하여 데이터를 수집, 저장, 관리


hadoop eco system


핵심 기술

분할 점령 : 데이터를 독립된 형태로 나누고 이를 병렬적으로 처리

	   하나의 결과로 만드는 것 



빅데이터 처리 기술 

CEP (Complex Event Processing)

여러 이벤트 소스로부터 발생한 이벤트를 대상으로 실시간으로 의미 있는 데이터를 추출하여
이에 대응하는 기능을 수행. 이벤트 데이터는 스크림 데이터로 대량으로 지속적으로 입력되는
데이터, 시간 순서가 중요하고 끝이 없는 데이터


batch 

처리해야 할 데이터를 일정 기간 또는 일정량 정리하여 처리하는 것

시스템에서는 처리의 대상이 되는 데이터를 일 단위나 월 단위마다 모아둔다 


빅데이터 일괄처리 플랫폼 (Hadoop)

하둡은 분산 파일시스템인 HDFS(하나의 파일처럼 보이지면 내부에는 여러 파일로 쪼개져서 작업 진행)
와 분선처리를 위한 맵리듀스로 구성됨



비정형 데이터 : 일일정한 규격이나 형태를 지닌 문자
데이터와 달리 그림이나 영상 문서처럼 형태와 구조가 다른 구조화 되지 않는 데이터


텍스트 마이닝(Text Mining) : 대규모 문서에서 의미있는 정보를 추출
분석 대상이 비구조적인 문서정보라는 점에서 데이터 마이닝과 차이
